SQL 高级语法教程(MySQL用法) 二

1.SQL 约束 (Constraints)：

约束用于限制加入表的数据的类型。
可以在创建表时规定约束（通过 CREATE TABLE 语句），或者在表创建之后也可以（通过 ALTER TABLE 语句）。
我们将主要探讨以下几种约束：

  • NOT NULL(非空)
  • UNIQUE(唯一键)
  • PRIMARY KEY(主键)
  • FOREIGN KEY(外键)
  • CHECK(限制)
  • DEFAULT(默认值)

1.NOT NULL(非空) 约束：

NOT NULL(非空) 约束强制列不接受 NULL(空) 值。
NOT NULL(非空) 约束强制字段始终包含值。这意味着，如果不向字段添加值，就无法插入新记录或者更新记录。

CREATE TABLE Persons
(
Id_P int NOT NULL,
LastName varchar(255) NOT NULL,
FirstName varchar(255),
Address varchar(255),
City varchar(255)
)

强制 “Id_P” 列和 “LastName” 列不接受 NULL(空) 值。

2.UNIQUE(唯一键) 约束：

UNIQUE(唯一键) 约束唯一标识数据库表中的每条记录。
UNIQUE(唯一键) 和 PRIMARY KEY(主键) 约束均为列或列集合提供了唯一性的保证。
PS：每个表可以有多个 UNIQUE(唯一键) 约束，但是每个表只能有一个 PRIMARY KEY(主键) 约束。
创建表时：

UNIQUE ( 列名称 )                       (创建 UNIQUE 约束)

CONSTRAINT 别名 UNIQUE ( 列1,列2 )      (如需别名 UNIQUE 约束，并定义多个列的 UNIQUE 约束)

表已创建时：

ALTER TABLE Persons
    ADD UNIQUE ( 列名称 )                       (创建 UNIQUE 约束)
或者：
    ADD CONSTRAINT 别名 UNIQUE ( 列1,列2 )      (如需别名 UNIQUE 约束，并定义多个列的 UNIQUE 约束)

撤销 UNIQUE 约束：

ALTER TABLE Persons
DROP INDEX 别名

3.PRIMARY KEY(主键) 约束：

PRIMARY KEY(主键) 约束唯一标识数据库表中的每条记录。

  1. 主键必须包含唯一的值。
  2. 主键列不能包含 NULL(空) 值。
  3. 每个表都应该有一个主键，并且每个表只能有一个主键。

创建表时：

PRIMARY KEY ( 列名称 )                       (创建 PRIMARY KEY 约束)

CONSTRAINT 别名 PRIMARY KEY ( 列1,列2 )      (如需别名 PRIMARY KEY 约束，并定义多个列的 PRIMARY KEY 约束)

表已创建时：

ALTER TABLE Persons
    ADD PRIMARY KEY ( 列名称 )                       (创建 PRIMARY KEY 约束)
或者：
    ADD CONSTRAINT 别名 PRIMARY KEY ( 列1,列2 )      (如需别名 PRIMARY KEY 约束，并定义多个列的 PRIMARY KEY 约束)

撤销 PRIMARY KEY 约束：

ALTER TABLE Persons
DROP PRIMARY KEY

4.FOREIGN KEY(外键) 约束：

一个表中的 FOREIGN KEY(外键) 指向另一个表中的 PRIMARY KEY(主键)。
创建表时：

FOREIGN KEY ( 列名称 ) REFERENCES 主键表 ( 主键列 )                      (创建 FOREIGN KEY 约束)

CONSTRAINT 别名 FOREIGN KEY ( 列名称 ) REFERENCES 主键表 ( 主键列 )      (如需别名 FOREIGN KEY 约束，并定义多个列的 FOREIGN KEY 约束)

表已创建时：

ALTER TABLE Persons
    ADD FOREIGN KEY ( 列名称 ) REFERENCES 主键表 ( 主键列 )                       (创建 FOREIGN KEY 约束)
或者：
    ADD CONSTRAINT 别名 FOREIGN KEY ( 列名称 ) REFERENCES 主键表 ( 主键列 )       (如需别名 FOREIGN KEY 约束，并定义多个列的 FOREIGN KEY 约束)

撤销 FOREIGN KEY 约束：

ALTER TABLE Persons
DROP FOREIGN KEY  ( 列名称 )

5.CHECK(限制) 约束：

用于限制列中的值的范围。
创建表时：

CHECK ( 列名称 > 0 )                                        (创建 CHECK 约束,规定 列 必须只包含大于 0 的整数。)

CONSTRAINT 别名 CHECK ( 列名称 > 0 AND 列名称 = 'xxx' )      (如需别名 CHECK 约束，并定义多个列的 CHECK 约束)

表已创建时：

ALTER TABLE Persons
    ADD CHECK ( 列名称 > 0 )                                        (创建 CHECK 约束,规定 列 必须只包含大于 0 的整数。)
或者：
    ADD CONSTRAINT 别名 CHECK ( 列名称 > 0 AND 列名称 = 'xxx' )      (如需别名 CHECK 约束，并定义多个列的 CHECK 约束)

撤销 UNIQUE 约束：

ALTER TABLE Persons
DROP CHECK 别名

6.DEFAULT(默认值) 约束：

用于向列中插入默认值。
创建表时：

列名称 数据类型 DEFAULT ( 列名称 )                          (创建 DEFAULT 约束)

OrderDate( 订单日期 ) date( 日期 ) DEFAULT GETDATE()      (使用类似 GETDATE() 这样的函数，DEFAULT 约束也可以用于插入系统值)

表已创建时：

ALTER TABLE Persons
ALTER 列名称 SET DEFAULT ' 内容 '                       (创建 DEFAULT 约束)

撤销 UNIQUE 约束：

ALTER TABLE Persons
ALTER 列名称 DROP DEFAULT

2.SQL CREATE INDEX 语句：

用于在表中创建索引。

    CREATE INDEX 索引名称           (创建索引)
    ON 表名称 ( 列名称 )
或者：
    CREATE UNIQUE INDEX 索引名称    (创建唯一索引)
    ON 表名称 ( 列名称 )

3.DROP 语句：

删除索引、表和数据库。

ALTER TABLE 表名称 DROP INDEX 索引名称   (删除表格中的索引)

DROP TABLE 表名称                       (删除表 (表的结构、属性以及索引也会被删除) )

DROP DATABASE 数据库名称                 (删除数据库)

TRUNCATE TABLE 表名称                   (仅仅删除表格中的数据)

4.SQL ALTER TABLE 语句：

用于在已有的表中添加、修改或删除列。

ALTER TABLE 表名称
    ADD 列名称 数据类型              (表中添加列)

    DROP COLUMN 列名称              (删除表中的列)

    MODIFY COLUMN 列名称 数据类型    (修改表中列的数据类型)

5.SQL AUTO INCREMENT 字段：

会在新记录插入表中时生成一个唯一的数字。
我们通常希望在每次插入新记录时，自动地创建主键字段的值。

列名称 数据类型 AUTO_INCREMENT               (默认 AUTO_INCREMENT 的开始值是 1)

ALTER TABLE 表名称 AUTO_INCREMENT=100       (AUTO_INCREMENT 的开始值是 100)

6.SQL VIEW（视图）：

视图是基于 SQL 语句的结果集的可视化的表。
CREATE VIEW 语法:

CREATE VIEW [ 视图名称 ] AS
SELECT 列名称
FROM 表名称
WHERE 条件

CREATE OR REPLACE VIEW(更新视图) 语法:

CREATE OR REPLACE VIEW [ 视图名称 ] AS
SELECT 列名称
FROM 表名称
WHERE 条件

撤销视图:

DROP VIEW view_name [ 视图名称 ]

7.SQL Date 函数:

日期。

函数 描述
NOW() 返回当前的日期和时间
CURDATE() 返回当前的日期
CURTIME() 返回当前的时间
DATE() 提取日期或日期/时间表达式的日期部分
EXTRACT() 返回日期/时间按的单独部分
DATE_ADD() 给日期添加指定的时间间隔
DATE_SUB() 从日期减去指定的时间间隔
DATEDIFF() 返回两个日期之间的天数
DATE_FORMAT() 用不同的格式显示日期/时间

Date 数据类型:
MySQL 使用下列数据类型在数据库中存储日期或日期/时间值：

  • DATE - 格式 YYYY-MM-DD
  • DATETIME - 格式: YYYY-MM-DD HH:MM:SS
  • TIMESTAMP - 格式: YYYY-MM-DD HH:MM:SS
  • YEAR - 格式 YYYY 或 YY

8.SQL NULL 值:

遗漏的未知数据。

列名称 IS NULL          (带有 NULL 值的记录)

列名称 IS NOT NULL      (不带有 NULL 值的记录)
